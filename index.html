# Firebase Setup Instructions ## Step 1: Add Your Firebase Config 1. Open `src/firebase.js` 2. Replace the placeholder config with YOUR config from Firebase Console 3. Find your config at: Firebase Console → Project Settings → Your apps → Config Replace this: ```javascript const firebaseConfig = { apiKey: "YOUR_API_KEY", // ← Replace these authDomain: "YOUR_PROJECT_ID.firebaseapp.com", projectId: "YOUR_PROJECT_ID", storageBucket: "YOUR_PROJECT_ID.appspot.com", messagingSenderId: "YOUR_SENDER_ID", appId: "YOUR_APP_ID" }; ``` ## Step 2: Update to Firebase Version The current `src/App.jsx` uses browser storage. **Two options:** ### Option A: Keep Browser Storage (Current Version) - Works immediately - Data can be lost - Good for testing ### Option B: Use Firebase (Recommended for Launch) - Requires Firebase setup complete - Permanent data storage - Better for production ## Step 3: Deploy ```bash # Push to GitHub git add . git commit -m "Add Firebase" git push # Vercel auto-deploys! ``` ## Testing Firebase 1. Create account on your site 2. Check Firebase Console → Authentication → Users (should see new user) 3. Check Firestore Database → Data (should see collections) ## Troubleshooting **"Firebase not defined"** - Make sure you added your config to `src/firebase.js` **"Permission denied"** - Check Firestore Rules are published **"User not created"** - Check Authentication is enabled for Email/Password
